How it works

From rollout to receipts.

No service to run. No new logging layer. It uses the data Codex already leaves on disk.

Codex writes rollout logs

Messages, commands, outputs, file edits, and token snapshots land in local JSONL sessions.

flightrec parses + matches

Claims are paired with nearby test and smoke-command evidence, including failures.

One HTML report

A complete, portable cockpit renders with its CSS, JavaScript, and SVG inline.

Everything stays on your machine.

Built by the thing it audits
Built with Codex on GPT-5.6 for OpenAI Build Week 2026 — and audited by itself: the first logs it ever analyzed were the sessions in which Codex was building it.
Field-tested
